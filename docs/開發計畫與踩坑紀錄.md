# 高手揪派 · LINE 預約系統 — 開發計畫與踩坑紀錄

> 適用對象：無程式底子、會用終端機的技術股東  
> 專案 repo：`https://github.com/julietbizlab-svg/gosu-chill-booking`  
> 最後更新：2026-06

---

## 一、專案目標

讓學員在 **LINE 官方帳號** 內一鍵預約課程，老師在 **Notion** 查看報名名單，無需記密碼。

| 角色 | 使用工具 |
|------|----------|
| 學員 | LINE → LIFF 預約頁 |
| 老師 | Notion 課表、學員資料、預約記錄 |
| 系統 | GitHub Pages（前端）+ Cloudflare Workers（後端） |

---

## 二、系統架構圖

```
學員手機 LINE
    ↓ 開啟 LIFF 連結
GitHub Pages（student-ui → docs/）
    ↓ LIFF 取得 userId、顯示名稱
Cloudflare Workers API（gosu-chill-api）
    ↓ Notion API
Notion 三張表：學員資料 / 課表 / 預約記錄
    ↓（選用）
LINE Messaging API → 剩 3 堂推播提醒
```

### 階段 1：前端 + LIFF + GitHub Pages

#### 1-1 建立專案結構

```
gosu-chill-booking/
├── student-ui/     ← 學員預約畫面（開發用）
├── docs/           ← 同步到這裡給 GitHub Pages 用
├── backend/        ← Cloudflare Workers 後端
└── scripts/sync-docs.sh
```

#### 1-2 LINE LIFF 設定

1. LINE Developers → Provider「飛」→ **預約系統 LIFF 專用**
2. 建立 LIFF App：
   - Endpoint URL：`https://julietbizlab-svg.github.io/gosu-chill-booking/`
   - Size：Full
3. 記下 **LIFF ID**：`2010530394-zbVNU8Pq`
4. **發布頻道**：Developing → **Published**（見踩坑 #8）
5. LIFF App 本身也要 **Published**

#### 1-3 設定 config.js

編輯 `student-ui/js/config.js`：

```javascript
window.GOSU_CONFIG = {
  LIFF_ID: "2010530394-zbVNU8Pq",
  API_BASE_URL: "https://gosu-chill-api.gosu-chill-book.workers.dev"
};
```

#### 1-4 部署 GitHub Pages

1. Repo → Settings → Pages → Source：**Deploy from branch**
2. Branch：`main`，資料夾：**`/docs`**
3. 每次改 `student-ui/` 後執行：

```bash
cd /Users/imac/project/gosu-chill-booking
bash scripts/sync-docs.sh
git add student-ui docs
git commit -m "更新預約畫面"
git push
```

4. 等 1～2 分鐘，手機關掉 LINE 預約頁重新開啟

### 階段 2：後端 + Notion

#### 2-1 建立 Notion 三張表

在 Notion 建立並 **連接 Integration**（每張表右上角 ··· → Connections）：

**① 學員資料**

| 欄位 | 類型 | 說明 |
|------|------|------|
| 預約編號 | 標題 | 填 LINE userId（U 開頭） |
| 學員姓名 | 文字 | |
| 剩餘堂數 | 數字 | |
| 到期日 | 日期 | |
| 狀態 | 選項 | `有效` / `待審` |
| 體驗贈送日 | 日期 | 系統可自動建立 |
| 系統記錄堂數 | 數字 | **勿手動改**，系統用 |
| 低堂數已提醒 | 勾選 | **勿手動改**，系統用 |

**② 課表（例如 2026年7月課表）**

| 欄位 | 類型 | 說明 |
|------|------|------|
| 課程名稱 | 標題 | 例：`椅子瑜伽｜下午`（見踩坑 #10） |
| 日期 | 日期 | |
| 時間 | 文字 | 例：`14:30~15:30` |
| 星期 | 選項 | 可選 |
| 備註 | 文字 | 停課請寫「老師進修停課」 |
| 老師 | 文字 | 系統可自動建立 |

**③ 預約記錄**

| 欄位 | 類型 |
|------|------|
| 預約編號 | 標題 |
| LINE userId | 文字 |
| 課程ID | 文字 |
| 狀態 | 選項：`已確認` / `已取消` |
| 學員姓名 | 文字 |
| 課程名稱 | 文字 |
| 上課時間 | 文字 |

#### 2-2 取得 Database ID

- 開啟資料庫網址，**32 字元英數**即 ID（可含連字號）
- ⚠️ 頁面 ID ≠ 資料庫 ID（見踩坑 #3）

#### 2-3 設定後端機密

```bash
cd backend
bash setup-dev-vars.sh
# 依提示貼上 NOTION_TOKEN 與三個 Database ID
```

驗證 Notion 連線：

```bash
bash test-notion.sh
```

#### 2-4 部署 Cloudflare Workers

```bash
cd backend
npx wrangler login
npx wrangler secret bulk .dev.vars
npx wrangler deploy
```

#### 2-5 驗收

1. 在 Notion 學員資料建立測試學員（預約編號 = LINE userId）
2. 手機 LINE 開預約連結 → 預約一堂課
3. 確認：堂數減少、預約記錄出現、狀態「已確認」

---

### 階段 3：進階功能

#### 3-1 老師查看視圖（Notion 手動）

1. 預約記錄 → **+ 新增視圖** → 表格 → 命名「老師查看」
2. 篩選：狀態 = 已確認
3. 排序：上課時間 升序
4. 顯示：學員姓名、課程名稱、上課時間、狀態

#### 3-2 課程名稱格式（自動）

- 下午課（12:00～17:59）：`椅子瑜伽｜下午`
- 晚上課（18:00 後）：`療癒陰瑜伽｜夜間`
- 老師名在「老師」欄，不在課程名稱裡

批次更新課表（若需要）：

```bash
cd backend
node scripts/label-course-times.mjs
```

#### 3-3 學員自動註冊

- 新學員第一次開預約頁 → 自動寫入 Notion 學員資料
- 老師只需設定 **剩餘堂數** 和 **到期日**

#### 3-4 體驗課（贈送第一堂，兩週有效）

在 Notion 學員資料：

| 欄位 | 值 |
|------|-----|
| 剩餘堂數 | 1 |
| 體驗贈送日 | 贈送當天 |
| 狀態 | 有效 |

#### 3-5 購買課程（自動延長 90 天）

老師在 Notion 把 **剩餘堂數加多**（例如 0→8）  
→ 學員下次開預約頁，系統自動把 **到期日** 設為 **當天+90 天**

#### 3-6 剩 3 堂 LINE 提醒

1. LINE Developers → 預約系統大工程 → Messaging API → 複製 **Channel access token**
2. 設定：

```bash
cd backend
npx wrangler secret put LINE_CHANNEL_ACCESS_TOKEN
# 貼上 token
```

## 五、踩過的坑（完整紀錄）

### 坑 #1：GitHub Pages 找不到 student-ui

**現象**：推送後網址 404  
**原因**：GitHub Pages 只支援根目錄或 `/docs`，不支援 `/student-ui`  
**解法**：`student-ui/` 同步到 `docs/`，用 `scripts/sync-docs.sh`

---

### 坑 #2：NOTION_TOKEN 貼成 Database ID

**現象**：API 錯誤 `token invalid`  
**原因**：把 32 字元資料庫 ID 當成 Token  
**解法**：Token 必須以 `ntn_` 或 `secret_` 開頭；用 `backend/test-notion.sh` 檢查

---

### 坑 #3：Database ID 抄成 Page ID

**現象**：查不到課程或學員  
**原因**：Notion 網址有時是「頁面」ID，不是「資料庫」ID  
**解法**：在資料庫內按 `...` → Copy link to view，或 API 查 `child_database`

---

### 坑 #4：Notion 欄位名稱與程式不一致

**現象**：讀不到日期、姓名、課程  
**原因**：程式預期 `上課日期`，實際是 `日期`；預期 `LINE userId` 欄，實際用 `預約編號`  
**解法**：`notion.js` 已做彈性對應；新建欄位請對照本文第四節

---

### 坑 #5：學員找不到 / 無法預約

**現象**：「找不到學員資料」  
**原因**：Notion「預約編號」未填該學員的 LINE userId（U 開頭）  
**解法**：從預約頁綠色狀態列複製編號貼到 Notion；現已支援自動註冊

---

### 坑 #6：Invalid Date / 課程顯示異常

**現象**：日期顯示 Invalid Date  
**原因**：時間格式 `14:30~15:30`，程式只預期 `14:30`  
**解法**：`formatDateText` 取 `~` 前段；後端統一處理

---

### 坑 #7：老師看不出誰預約了

**現象**：預約記錄只有 UUID，看不懂  
**原因**：舊版只寫技術 ID  
**解法**：預約時寫入學員姓名、課程名稱、上課時間；建立「老師查看」視圖

---

### 坑 #8：其他 LINE 帳號 400 Bad Request

**現象**：`This channel is now developing status. User need to have developer role`  
**原因**：LIFF 頻道還在 **Developing**，只有測試者能用  
**解法**：
- 測試期：LINE Developers → Members → 加 Tester
- 正式上線：LINE Login 頻道 + LIFF App 都改 **Published**

---

### 坑 #9：學員資料改了，預約記錄沒更新

**現象**：姓名在學員表是對的，預約記錄仍是舊名  
**原因**：兩張表獨立，預約當下寫死的  
**解法**：登入時自動同步該學員所有預約記錄的姓名與預約編號

---

### 坑 #10：課程名稱格式混亂

**現象**：`椅子瑜伽下午｜佳貞老師` 難讀  
**原因**：手動命名不統一  
**解法**：統一為 `椅子瑜伽｜下午`，老師名放「老師」欄；執行 `label-course-times.mjs`

---

### 坑 #11：停課日仍顯示可預約

**現象**：7/10 老師進修停課，月曆仍可點預約  
**原因**：Notion 只填備註、課名時間留空，舊程式仍當成課程  
**解法**：課名或時間為空、或備註含「停課」→ 不顯示

---

### 坑 #12：顯示「LINE學員」或錯誤姓名

**現象**：問候語不對；Notion 姓名錯誤  
**原因**：後端測試時寫入錯誤名稱；或登入未傳 displayName  
**解法**：用 POST 傳 LINE 名稱；登入時若與 Notion 不同則自動更新

---

### 坑 #13：改程式後手機沒變

**現象**：明明 deploy 了，手機還是舊畫面  
**原因**：
- 前端要 `git push` 才更新 GitHub Pages
- 後端 `wrangler deploy` 才更新 API
- LINE 內建瀏覽器快取  
**解法**：push 後等 1～2 分鐘，**完全關閉** LINE 預約頁再開

---

### 坑 #14：git 作者警告

**現象**：`fix the identity used for this commit`  
**原因**：git user.name / email 未設定  
**解法**（可選）：

```bash
git config --global user.name "你的名字"
git config --global user.email "你的信箱"
```

不影響系統運作，只影響 commit 顯示名稱。

## 六、日常維運手冊

### 新增一堂課（Notion 課表）

1. 新增一列
2. 課程名稱：`流動瑜伽｜佳貞老師`（系統會自動改成 `流動瑜伽｜下午` 或 `｜夜間`）
3. 日期、時間填好
4. 不必手動加「下午/夜間」

### 贈送體驗課

1. 學員資料 → 找到該學員（或等他自己登入自動建立）
2. 剩餘堂數 = 1，體驗贈送日 = 今天

### 學員購買方案

1. 在學員資料把 **剩餘堂數** 加上去（例如 +8）
2. 學員下次開預約頁 → 到期日自動 +90 天

### 停課

- **做法 A（建議）**：日期填好，課程名稱與時間**留空**，備註寫「老師進修停課」
- **做法 B**：直接刪除該列
